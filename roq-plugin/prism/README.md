# Quarkus Roq Plugin — Prism.js

A [Roq](https://iamroq.dev/) plugin that adds [Prism](https://prismjs.com/) syntax
highlighting to a Roq site. The plugin assembles a single `prism.js` + `prism.css`
bundle at build time from the languages and theme you select, and serves them as
static resources.

The bundle is generated by reading the language and theme files straight out of
the `org.mvnpm:prismjs` JAR — no esbuild step, no Node dependency.

## Install

Add the dependency to your Roq site:

```xml
<dependency>
    <groupId>io.quarkiverse.roq</groupId>
    <artifactId>quarkus-roq-plugin-prism</artifactId>
    <version>1.0.0-SNAPSHOT</version>
</dependency>
```

## Configure

In `application.properties`, list the Prism languages you want bundled. The
`languages` property is **required** — an empty or missing value fails the
build.

```properties
quarkus.roq.prism.languages=java,bash,properties,yaml,markup-templating,json
quarkus.roq.prism.theme=tomorrow
```

| Property                      | Description                                                                                                                                                                                                                                                                                                      | Default    |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------- |
| `quarkus.roq.prism.languages` | Comma-separated list of canonical language names from [Prism's `components.json`](https://github.com/PrismJS/prism/blob/master/components.json). See [Supported languages](https://prismjs.com/#supported-languages). Transitive `require` prerequisites are added automatically (e.g. `java` pulls in `clike`). | *required* |
| `quarkus.roq.prism.theme`     | One of `default`, `coy`, `dark`, `funky`, `okaidia`, `solarizedlight`, `tomorrow`, `twilight`. Preview by picking a theme under [Themes](https://prismjs.com/index.html#theme) on prismjs.com and scrolling to the [Examples](https://prismjs.com/index.html#examples) section.                                  | `default`  |

## Use

Insert the `{#prism /}` Qute tag wherever you want the bundle injected — usually
in your base layout's `<head>`:

```html
<head>
    ...
    {#prism /}
    {#bundle /}
</head>
```

The tag emits a `<link>` for the theme CSS and a deferred `<script>` for the
language bundle. The script auto-runs `Prism.highlightAll()` once the DOM is
parsed, so any `<pre><code class="language-X">…</code></pre>` produced by your
Markdown/AsciiDoc renderer gets highlighted.

## How it works

At build time, `RoqPluginPrismProcessor`:

1. Reads `quarkus.roq.prism.languages` and resolves the transitive `require`
   graph from `components.json`.
2. Concatenates `prism-core.min.js` followed by each requested language's
   `prism-<name>.min.js`, then appends the auto-highlight trigger.
3. Reads the requested theme's `prism[-<theme>].min.css`.
4. Emits both as `GeneratedStaticResourceBuildItem`s at
   `/static/bundle/prism.js` and `/static/bundle/prism.css`.

Because the bundle is assembled from already-minified Prism files, the output
should be small.
