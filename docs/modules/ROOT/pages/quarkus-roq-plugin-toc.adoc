= Quarkus Roq TOC Plugin

The Roq TOC plugin generates a table of contents from the headings in your Markdown and AsciiDoc pages.
It parses rendered HTML at runtime, extracts headings with IDs, and exposes the result as structured data or a ready-to-use HTML navigation block in your Qute templates.

NOTE: The TOC plugin is not included in the `io.quarkiverse.roq:quarkus-roq` aggregate extension. Add the dependency described in <<installation>> to use it.

[id="understand-toc"]
== Understand the TOC plugin

The plugin works by parsing the rendered HTML content of each page with https://jsoup.org/[JSoup].
It extracts all headings (`h1`–`h6`) that have an `id` attribute, which both the Markdown and AsciiDoc renderers generate automatically.

The extracted headings are organized into a nested hierarchy based on their levels.
For example, an `h3` following an `h2` becomes a child of that `h2` in the TOC tree.

Because the plugin operates on rendered HTML rather than raw source, it works consistently with both Markdown and AsciiDoc content without any special configuration.

All HTML output is XSS-escaped.

[id="installation"]
== Add a table of contents to your pages

=== Add the dependency

Add the TOC plugin to your project's `pom.xml`:

[source,xml,subs=attributes+]
----
<dependency>
    <groupId>io.quarkiverse.roq</groupId>
    <artifactId>quarkus-roq-plugin-toc</artifactId>
    <version>{project-version}</version>
</dependency>
----

=== Use the pre-rendered HTML block

To display a TOC with default markup, add `\{page.tocHtml}` to your layout template:

[source,html]
----
{page.tocHtml}

{#insert /}
----

This produces a `<nav class="roq-toc">` element containing a nested `<ul>` list with anchor links to each heading:

[source,html]
----
<nav class="roq-toc">
<ul>
<li><a href="#introduction">Introduction</a></li>
<li><a href="#getting-started">Getting Started</a>
<ul>
<li><a href="#prerequisites">Prerequisites</a></li>
<li><a href="#installation">Installation</a></li>
</ul>
</li>
</ul>
</nav>
----

Style the TOC with CSS using the `.roq-toc` class selector.

=== Use structured data for custom rendering

For full control over the TOC markup, iterate over the `\{page.toc}` list:

[source,html]
----
{#if page.toc.size > 0}
<aside class="my-toc">
  <h2>On this page</h2>
  <ul>
  {#for entry in page.toc}
    <li>
      <a href="#{entry.id}">{entry.title}</a>
      {#if entry.children.size > 0}
      <ul>
        {#for child in entry.children}
        <li><a href="#{child.id}">{child.title}</a></li>
        {/for}
      </ul>
      {/if}
    </li>
  {/for}
  </ul>
</aside>
{/if}
----

[id="reference"]
== Template API reference

=== Template extensions on `Page`

[cols="1,1,3"]
|===
| Expression | Return type | Description

| `\{page.toc}`
| `List<TocEntry>`
| A hierarchical list of TOC entries extracted from the page content. Returns an empty list if the page has no headings with IDs.

| `\{page.tocHtml}`
| `RawString`
| A pre-rendered `<nav class="roq-toc">` HTML block. Returns an empty string if the page has no headings with IDs.
|===

=== `TocEntry` properties

[cols="1,1,3"]
|===
| Property | Type | Description

| `id`
| `String`
| The heading's HTML `id` attribute, used as the anchor fragment.

| `title`
| `String`
| The heading's text content.

| `level`
| `int`
| The heading level (1–6).

| `children`
| `List<TocEntry>`
| Nested child entries (headings at a deeper level).
|===
