= Roq Plugin OG Card

The OG Image plugin generates 1200×630 PNG social preview cards from Qute SVG templates, injects `og-image` frontmatter so `{#seo /}` emits `og:image` and `twitter:card=summary_large_image`, and registers PNG paths with the Roq Generator for static output.

== Dependency

[source,xml]
----
<dependency>
  <groupId>io.quarkiverse.roq</groupId>
  <artifactId>quarkus-roq-plugin-og-card</artifactId>
</dependency>
----

== How it works

At build time the deployment processor:

. Selects pages from configured collections and/or explicit include paths
. Reads page frontmatter (`title`, `description`, and optional keys such as `kicker` and `eyebrow`) to build card data
. Renders each card to PNG with Apache Batik and publishes the bytes as static files via `RoqFrontMatterStaticFileBuildItem`
. Injects `og-image`, `og-image-width`, `og-image-height`, and `og-image-alt` into page frontmatter

When present, the plugin overrides the bundled `{#seoImage /}` tag with a version that checks `page.data.og-image` first, then falls back to the standard `page.image` behaviour.

== Configuration

All properties use the prefix `quarkus.roq.plugin.og-card` and are fixed at build time.

[source,properties]
----
quarkus.roq.plugin.og-card.collections=posts
quarkus.roq.plugin.og-card.include-paths=/
quarkus.roq.plugin.og-card.exclude-paths=/posts/tag/
quarkus.roq.plugin.og-card.output-prefix=/og
quarkus.roq.plugin.og-card.collection-output-segment=:collections
quarkus.roq.plugin.og-card.width=1200
quarkus.roq.plugin.og-card.height=630
quarkus.roq.plugin.og-card.skip-if-image-set=true
quarkus.roq.plugin.og-card.site-name=My Site
quarkus.roq.plugin.og-card.template=og-card/default-card.svg
----

[cols="2,1,3"]
|===
| Property | Default | Purpose

| `collections`
| *(empty)*
| Roq collection ids for which OG images are generated (e.g. `posts` → `/og/posts/{slug}.png`). At least one of `collections` or `include-paths` must be set.

| `include-paths`
| *(empty)*
| Explicit normal page paths to include (e.g. `/` → `/og/index.png`, `/about/` → `/og/about.png`).

| `exclude-paths`
| *(empty)*
| When using `collections`, page path prefixes to skip (e.g. `/posts/tag/` for auto-generated tag archives). Not applied to `include-paths`.

| `output-prefix`
| `/og`
| Base URL path for generated PNG files.

| `collection-output-segment`
| `:collections`
| Folder name for collection documents. `:collection` is replaced with the collection id; `:collections` appends `s` when the id does not already end with `s` (`guide` → `guides`).

| `width` / `height`
| `1200` / `630`
| Output PNG dimensions in pixels (passed to the Batik transcoder).

| `skip-if-image-set`
| `true`
| When `true`, pages that already declare `image`, `img`, or `picture` frontmatter are skipped.

| `site-name`
| `Site`
| Site branding on the card and fallback for `og-image-alt` text.

| `template`
| `og-card/default-card.svg`
| Qute template id for the SVG card (site templates under `templates/` override bundled defaults).
|===

== Headless mode

PNG rendering uses Apache Batik, which relies on Java2D/AWT for text and graphics—even though the plugin does not use Swing, JavaFX, or any desktop UI directly.

On macOS and other environments with a display, the JVM defaults to *non-headless* AWT (`java.awt.headless=false`). When Batik rasterizes a card, you may briefly see a Java process appear in the dock or task switcher. In CI, containers, and generator batch runs, set headless mode explicitly:

[source,properties]
----
# application.properties — dev mode and quarkus:run
quarkus.args=-Djava.awt.headless=true
----

Or export an environment variable before Maven or your container entrypoint:

[source,shell]
----
export JAVA_TOOL_OPTIONS=-Djava.awt.headless=true
QUARKUS_ROQ_GENERATOR_BATCH=true mvn package quarkus:run
----

Headless rendering still needs fonts available to the JVM. The bundled template references `Arial`; on minimal Linux images install a font package or switch the template to a font present in the image.

== SVG card template

Add a Qute SVG template under `templates/og-card/` and point `quarkus.roq.plugin.og-card.template` at its id (path relative to `templates/`, e.g. `og-card/my-card.svg`).

At render time the plugin passes a single `card` data object to the template:

[source,java]
----
record OgCardData(String title, String description, String siteName,
                  String kicker, String eyebrow, int width, int height)
----

Use `{card.width}` and `{card.height}` in the SVG root so the template stays aligned with `quarkus.roq.plugin.og-card.width` and `.height`. Title and description are XML-escaped before render.

Bundled default: `og-card/default-card.svg` in the extension jar.

=== How frontmatter feeds the template

Page frontmatter (FMT) is the input; the plugin maps selected keys into `card` fields at build time. Config properties fill in site-wide defaults and dimensions.

[cols="1,1,2"]
|===
| Source | `card` field | Notes

| `title` frontmatter
| `card.title`
| Falls back to `quarkus.roq.plugin.og-card.site-name` when absent. Truncated to 120 characters.

| `description` frontmatter
| `card.description`
| Falls back to `title` when absent. Truncated to 200 characters.

| `kicker` frontmatter
| `card.kicker`
| Optional short label above the title (e.g. section or series name).

| `eyebrow` frontmatter
| `card.eyebrow`
| Optional secondary label; not used by the bundled default template but available in custom templates.

| `quarkus.roq.plugin.og-card.site-name`
| `card.siteName`
| Shown in the card footer and used when building alt text.

| `quarkus.roq.plugin.og-card.width` / `.height`
| `card.width` / `card.height`
| Also passed to the Batik PNG transcoder.
|===

Pages that declare `image`, `img`, or `picture` are skipped when `skip-if-image-set=true` (the default), because Roq treats those keys as static assets under `images/`, not generated PNG routes.

After a page is selected, the plugin *injects* additional frontmatter keys used by `{#seo /}`—these are outputs, not template inputs:

* `og-image` — site-relative PNG path (e.g. `/og/posts/welcome-to-roq.png`)
* `og-image-width` / `og-image-height` — dimensions for Open Graph meta tags
* `og-image-alt` — alt text for `twitter:image:alt`

=== Example: page frontmatter and template

Given this collection post at `content/posts/welcome-to-roq.md`:

[source,yaml]
----
---
title: Welcome to Roq
description: Your first steps with Roq static sites.
kicker: Getting started
collection: posts
---
----

…and this configuration:

[source,properties]
----
quarkus.roq.plugin.og-card.collections=posts
quarkus.roq.plugin.og-card.exclude-paths=/posts/tag/
quarkus.roq.plugin.og-card.site-name=My Site
quarkus.roq.plugin.og-card.template=og-card/my-card.svg
----

…the build registers `/og/posts/welcome-to-roq.png`, injects `og-image` metadata into the page, and renders the card from a template like:

[source,xml]
----
<svg xmlns="http://www.w3.org/2000/svg"
     width="{card.width}" height="{card.height}"
     viewBox="0 0 {card.width} {card.height}">
  <rect width="100%" height="100%" fill="#0b1220"/>
  {#if card.kicker}
  <text x="72" y="88" fill="#78a9ff" font-family="Arial, sans-serif" font-size="18">
    {card.kicker}
  </text>
  {/if}
  <text x="72" y="170" fill="#ffffff" font-family="Arial, sans-serif" font-size="52" font-weight="700">
    {card.title}
  </text>
  <text x="72" y="250" fill="#c6c6c6" font-family="Arial, sans-serif" font-size="28">
    {card.description}
  </text>
  <text x="72" y="582" fill="#8d8d8d" font-family="Arial, sans-serif" font-size="22">
    {card.siteName}
  </text>
</svg>
----

The resulting PNG shows *Getting started* as the kicker, *Welcome to Roq* as the title, the description underneath, and *My Site* in the footer. Browse `/og/posts/welcome-to-roq.png` in dev mode, or fetch it during generator batch to write `target/roq/og/posts/welcome-to-roq.png`.

== Static site output

Run the generator batch to emit PNGs alongside HTML:

[source,shell]
----
JAVA_TOOL_OPTIONS=-Djava.awt.headless=true \
  QUARKUS_ROQ_GENERATOR_BATCH=true mvn package quarkus:run
----

PNG files appear under `target/roq/og/`.

In dev mode, browse `/og/posts/my-page.png` directly.

== Frontmatter keys

[cols="1,2"]
|===
| Key | Purpose

| `title`, `description`
| Card content; mapped to `card.title` and `card.description` in the SVG template.

| `kicker`, `eyebrow`
| Optional labels mapped to `card.kicker` and `card.eyebrow`.

| `image`, `img`, `picture`
| When present and `skip-if-image-set=true`, the page is excluded from OG image generation.

| `og-image`
| Site-relative PNG path injected by the plugin (e.g. `/og/posts/welcome-to-roq.png`).

| `og-image-width` / `og-image-height`
| Dimensions for `og:image:width` / `og:image:height` meta tags.

| `og-image-alt`
| Alt text for `twitter:image:alt`.
|===

Do not use the standard `image` frontmatter key for generated cards — Roq resolves `image` as a static file under `images/`.
